# vibe-business-data-models

Industry data models auto-generated by the **[vibe-modelling-agent](https://github.com/amralieg/vibe-modelling-agent)** — an LLM-powered pipeline that produces UC-ready data models, metric views, sample data, and DBML diagrams from a one-paragraph business description.

Each industry root folder contains the **two flavours** the agent produces:

- **`ecm_v1/` — Expanded Coverage Model** — comprehensive, audit-grade model used as the source of truth.
- **`mvm_v1/` — Minimum Viable Model** — shrunk, demo-ready model derived from the ECM (typically 40-60% of the ECM in product count, retaining the most-used entities and FK paths).

Generated with **agent v0.7.1** running on Databricks Serverless (GCP).

## Repository structure

```
<industry>/
├── readme.md                    # Industry-level summary (counts, vibe, generation metadata)
├── ecm_v1/
│   ├── readme.md                # ECM-specific summary
│   ├── model.json               # Full agent model (single source of truth)
│   ├── diagram/                 # DBML + auto-generated ER diagrams
│   ├── docs/                    # Per-domain markdown docs
│   ├── metrics/                 # Metric view SQL (one .sql per metric view)
│   ├── ontology/                # Tag taxonomy + classification ontology
│   ├── samples/                 # Realistic sample-data CSVs (sampled from each table)
│   ├── schemas/                 # Per-table DDL (CREATE TABLE / CREATE VIEW)
│   └── vibes/                   # next_vibes.txt — auto-generated improvement priorities for v2
└── mvm_v1/
    └── (same structure as ecm_v1)
```

## How models are generated

1. **Vibe-modelling-agent** receives a business description (e.g. "Agriculture — primary production, supply chain, finance, sustainability ...").
2. It runs an 8-stage pipeline (domain generation → product generation → attribute generation → linking → semantic dedup → naming → finalize → metric views) using a per-stage LLM ensemble + judge.
3. The output is saved to a Databricks workspace folder and a Unity Catalog volume.
4. An orchestrator pushes each completed industry into this repo as its own commit.

See the [vibe-modelling-agent README](https://github.com/amralieg/vibe-modelling-agent) for the full pipeline architecture.
